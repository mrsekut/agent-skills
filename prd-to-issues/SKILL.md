# PRDからタスクへの変換

Tracer Bullets方式の垂直スライスを使用して、PRDを独立して着手可能なタスクに分割します。

## プロセス

### 1. PRDを特定する

ユーザーにPRDの場所を確認します（`.specs/{project}/1-prd.md`など）。

PRDがまだコンテキストウィンドウにない場合は、ファイルを読み込みます。

### 2. コードベースを調査する（任意）

まだコードベースを調査していない場合は、コードの現在の状態を把握するために調査を行います。

### 3. 垂直スライスを起案する

PRDを**Tracer Bullets**方式のタスクに分割します。各タスクは、1つのレイヤーの水平スライスではなく、すべての統合レイヤーをエンドツーエンドで貫通する薄い垂直スライスです。

<vertical-slice-rules>
- 各スライスは、すべてのレイヤー（スキーマ、API、UI、テスト）を貫通する狭いが完全なパスを提供する
- 完了したスライスは、それ単体でデモまたは検証が可能である
- 少数の厚いスライスよりも、多数の薄いスライスを優先する
</vertical-slice-rules>

### 4. ユーザーに確認する

提案した分割内容を番号付きリストで提示します。各スライスについて以下を表示します：

- **タイトル**: 短い説明的な名前
- **ブロッカー**: 先に完了する必要がある他のスライス（ある場合）
- **対応するユーザーストーリー**: PRDのどのユーザーストーリーに対応するか

ユーザーに以下を確認します：

- 粒度は適切か？（粗すぎる / 細かすぎる）
- 依存関係は正しいか？
- スライスの統合や追加分割が必要か？

ユーザーが分割内容を承認するまで繰り返します。

### 5. タスクリストを作成する

承認された各スライスを、以下のテンプレートでmarkdownファイルに記録します。

sddワークフロー内で使用している場合は`.specs/{project}/3-issues.md`に出力します。

<task-template>
# タスク一覧

PRD: {PRDファイルへの参照}

## 1. {タスクタイトル}

- **ブロッカー**: なし / タスク{N}

### 構築する内容

この垂直スライスの簡潔な説明。レイヤーごとの実装ではなく、エンドツーエンドの動作を記述します。

### 受け入れ基準

- [ ] 基準1
- [ ] 基準2
- [ ] 基準3

### 対応するユーザーストーリー

- ユーザーストーリー3
- ユーザーストーリー7

---

## 2. {タスクタイトル}

...
</task-template>
