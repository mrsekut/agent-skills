# ディープモジュール

「A Philosophy of Software Design」より：

**ディープモジュール** = 小さなインターフェース + 多くの実装

```
┌─────────────────────┐
│   小さなインターフェース   │  ← 少ないメソッド、シンプルなパラメータ
├─────────────────────┤
│                     │
│                     │
│  深い実装            │  ← 複雑なロジックを隠蔽
│                     │
│                     │
└─────────────────────┘
```

**シャローモジュール** = 大きなインターフェース + 少ない実装（避けるべき）

```
┌─────────────────────────────────┐
│       大きなインターフェース           │  ← 多くのメソッド、複雑なパラメータ
├─────────────────────────────────┤
│  薄い実装                        │  ← ただ通過するだけ
└─────────────────────────────────┘
```

インターフェースを設計する際に問うべきこと：

- メソッドの数を減らせるか？
- パラメータをシンプルにできるか？
- より多くの複雑さを内部に隠蔽できるか？
